# Docker configurations for Athenz integration tests

The test certificate authority (CA) and key pairs are generated by the
[`create-self-signed-certs.sh`](https://github.com/AthenZ/athenz/blob/a76b62563337ed579a990becb61f664cc3121e1f/docker/setup-scripts/self-signed-certificates.sh)
and
[`acceptance-test.sh`](https://github.com/AthenZ/athenz/blob/a76b62563337ed579a990becb61f664cc3121e1f/docker/deploy-scripts/acceptance-test.sh)
scripts. Please refer to the
[Athenz Docker documentation](https://github.com/AthenZ/athenz/blob/master/docker/docs/try-out-Athenz-with-self-signed-CA.md)
for more details on how to set up a self-signed CA for Athenz.

## Certificate validity

All fixture certificates are issued for 100 years (`-days 36500`). Beware that the upstream
[`sample/zts/create-self-signed-certs.sh`](https://github.com/AthenZ/athenz/blob/a76b62563337ed579a990becb61f664cc3121e1f/docker/sample/zts/create-self-signed-certs.sh)
script hardcodes `-days 364` for the ZTS server certificate;
the certificate generated that way expired on 2026-07-16 and broke CI, so it has been reissued with
`-days 36500` like the others. `AthenzCertificateExpiryTest` fails 30 days before any fixture
certificate expires.

## Renewing a certificate

The CA private keys live in this fixture (`certs/CAs/athenz_ca.pem`, `service_ca.pem` and `user_ca.pem`
each contain the CA key pair, and the Intermediate CA sits inside
`zts/var/certs/zts_signer_keystore.pkcs12`; every keystore password is `athenz`), so any certificate
can be reissued while keeping its key pair. For example, the ZTS server certificate:

```bash
cd athenz/src/test/resources/docker
# Extract the existing private key. (-legacy is only needed for keystores that still use RC2 encryption.)
openssl pkcs12 -legacy -in zts/var/certs/zts_keystore.pkcs12 -nocerts -nodes -passin pass:athenz |
    openssl pkey -out zts_key.pem
openssl req -new -key zts_key.pem -subj "/C=US/O=Athenz/CN=Sample Self Signed ZTS" -out zts.csr
cat > ext.cnf <<'EOF'
basicConstraints=critical,CA:FALSE
keyUsage=critical,digitalSignature,nonRepudiation,keyEncipherment
subjectAltName=DNS:localhost,DNS:athenz-zts-server,IP:127.0.0.1
subjectKeyIdentifier=hash
authorityKeyIdentifier=keyid,issuer
EOF
openssl x509 -req -in zts.csr -CA certs/CAs/athenz_ca_cert.pem -CAkey certs/CAs/athenz_ca.pem \
    -CAcreateserial -days 36500 -sha256 -extfile ext.cnf -out zts_cert.pem
openssl pkcs12 -export -in zts_cert.pem -inkey zts_key.pem -name 1 -passout pass:athenz \
    -out zts/var/certs/zts_keystore.pkcs12
```

Service certificates (`certs/foo-service`, `certs/test-service`) are issued by the Intermediate CA with
`extendedKeyUsage=clientAuth` and their `*.athenz.cloud` SANs, and bundle the Intermediate CA
certificate after the leaf. Mirror the existing certificate when renewing:
`openssl x509 -in certs/<service>/cert.pem -noout -text`.
