# v1.10.0

## 🌟 New features

- A new module `armeria-sangria` that integrates with [Sangria GraphQL](https://sangria-graphql.github.io) is
  introduced to let you serve a GraphQL request in Scala. #3703 #3704
  ```scala
  val schema: Schema[CharacterRepo, Unit] = Schema(Query)
  Server.builder()
        .service("/graphql",
            SangriaGraphqlService.builder(schema, new CharacterRepo)
                                 .enableTracing(true)
                                 .maxQueryDepth(10)
                                 .build())
  ```
- You can now configure [WebClient](type) to automatically follow redirections. #2489 #3641
  ```java
  WebClient.builder()
           .followRedirects()
           .build();
  // Customize redirection policy
  RedirectConfig config = RedirectConfig.builder()
                                        .maxRedirects(10)
                                        .allownDomains("foo.com", "bar.com")
                                        .build();
  WebClient.builder("https://example.com")
           .followRedirects(config)
           .build();
  ```
- You can now recover a failed [HttpResponse](type) with a fallback [HttpResponse](type). It would be useful
  for handling an error of [HttpResponse](type) in a decorator. #3674
  ```java
  HttpResponse response = 
      HttpResponse.ofFailure(new IllegalStateException("Oops..."));
  // The failed HttpResponse will be recovered by the fallback function.
  HttpResponse recovered = 
      response.recover(cause -> HttpResponse.of("Fallback"));
  assert recovered.aggregate().join().contentUtf8().equals("Fallback");
  ```
- You can now resume a failed [StreamMessage](type) with a fallback [StreamMessage](type). #3674
  ```java
  DefaultStreamMessage<Integer> stream = new DefaultStreamMessage<>();
  stream.write(1);
  stream.write(2);
  stream.close(new IllegalStateException("Oops..."));
  StreamMessage<Integer> resumed = 
      stream.recoverAndResume(cause -> StreamMessage.of(3, 4));
  assert resumed.collect().join().equals(List.of(1, 2, 3, 4));
  ```
- You can now transform an error of [StreamMessage](type) into another using
  [StreamMessage#mapError(Function)](type). #3668
  ```java
  StreamMessage stream = 
      StreamMessage.aborted(ClosedStreamException.get());
  StreamMessage transformed = stream.mapError(ex -> {
    if (ex instanceof ClosedStreamException) {
      return new IllegalStateException(ex);
    } else {
      return ex;
    }
  }); 
  ```
- You can now automatically encode an object into JSON for a response payload using
  [HttpResponse#ofJson(Object)](type). #3662
  ```java
  MyObject myObject = ...;
  HttpResponse.ofJson(myObject);
  MyError myError = ...;
  HttpResponse.ofJson(HttpStatus.INTERNAL_SERVER_ERROR, myError);
  ```
- You can now fluently inject `io.grpc.ServerInterceptor`s using
  [GrpcServiceBuilder#intercept(Iterable)](type).
  ```java
  GrpcService.builder()
             .addService(myService)
             .intercept(myInterceptror)
             .build();
  ``` 
- You can now easily add `Accept` headers with [RequestHeadersBuilder#accept(Iterable)](type). #3704
  ```java
  RequestHeaders.builder()
                .accept(MediaType.JSON)
                .accept(MediaType.PLAIN_TEXT);
  ```
- You can now initiate graceful connection shutdown using
  [ServiceRequestContext#initiateConnectionShutdown()](type). #3516 #3715
- You can now specify the default `ObjectMapper` using [JacksonObjectMapperProvider](type). #3728
  ```java
  public class MyObjectMapperProvider 
      implements JacksonObjectMapperProvider {
  
    @Override
    public ObjectMapper newObjectMapper() {
      return JsonMapper
         .builder()
         .addModules(new KotlinModule())
         .enable(DeserializationFeature.FAIL_ON_NULL_FOR_PRIMITIVES)
         .build();
    }
  }
  ```
- You can now serve `application/grpc-web+proto` and `application/grpc-web-text+proto`
  [protocols](https://github.com/grpc/grpc/blob/master/doc/PROTOCOL-WEB.md#protocol-differences-vs-grpc-over-http2)
  using [AbstractUnaryGrpcService](type). #3638 #3716
- gRPC trailers are available in the [RequestContext](type) for unary gRPC server and client. #3724 #3739
  ```java
  UnaryGrpcClient client = ...;
  try (ClientRequestContextCaptor captor = Clients.newContextCaptor()) {
      client.execute("/com.example.MyService/UnaryCall", request); 
      ClientRequestContext ctx = captor.get();
      HttpHeaders trailers = GrpcWebTrailers.get(ctx); // 👈👈👈
  }
  ```
- You can now compress and decompress content with [Brotli](https://www.brotli.org/) when using
  [EncodingService](type), [DecodingService](type) and [DecodingClient](type). #3544 #3686
- You can now create an [HttpResponseException](type) with a cause. #3674
- You can now easily capture [ServiceRequestContext](typeplural) using [ServerExtension](type) in tests. #3648
- You can now suppress the inconsistent Netty version warning by specifying
  `-Dcom.linecorp.armeria.warnNettyVersions=false`. #3572 #3766

## 📃 Documentation

- You can now learn how to write a REST service with Armeria by walking through the
  [brand-new tutorials](/tutorials). #3420
  - Special thanks to @freevie who volunteered for this.

## 📈 Improvements

- [HttpHeaders](type) now properly caches well-known HTTP headers' value objects. #3711 #3714
- [ConcurrencyLimitTimeoutException](type) is now raised when a request times out
  in [ConcurrencyLimitingClient](type). #3681
- [OAuth2AuthorizationGrant](type) no longer uses locks for better performance. #3571 #3618

## 🛠️ Bug fixes

- You no longer see an `IllegalArgumentException` when [HttpRequest#aggregate()](type)
  and [HttpResponse#aggregate()](type) fail with an exception. #3676 #3684 #3687
- You no longer see an incomplete [RequestLog](type) when an [HttpStatusException](type)
  is raised. #3719 #3674
- You no longer see a `NullPointerException` when an HTTP/1 connection of a client is closed. #3729 #3747
- You no longer see a `NullPointerException` when a [StreamMessage](type) is aborted. #3731
- No more unintended leakage of sensitive information when logging a [RequestLog](type). #3758
- A request is now correctly reset when [ServiceRequestContext#cancel()](type) is called. #3674
- You no longer see a `ClassNotFoundException` in the shaded fastutil. #3713
- Armeria now accepts harmless yet illegal characters in a request path for HTTP/1 connections, like it did for
  HTTP/2. #3778
- You no longer see an [EmptyEndpointGroupException](type) when [Endpoint](typeplural) are completely switched
  to new [Endpoint](typeplural) in [HealthCheckedEndpointGroup](type). #3637
- You can now correctly monitor a [BlockingTaskExecutor](type) with Micrometer. #3710 #3718
- [BlockingTaskExecutor](type)'s idle threads are now properly terminated after timeout. #3718
- You no longer see a `ClassNotFoundException` when a [TransportType](type) is initialized. #3717
- The second [RetryRuleWithContent](type) is now properly applied to decide to retry or not when
  using [RetryingClient](type). #3720
- `requestDurationNanos`, `responseDurationNanos` and `totalDurationNanos` of a child log are properly
  calculated. #3749
- An `IllegalArgumentException` raised in annotated service will be correctly logged by
  [LoggingService](type). #3674
- You no longer see [ClosedSessionException](type) when a large content is sent to an invalid path. #3674
- [ServerBuilder#localPort(int,Iterable)](type) correctly binds both IPv4 and IPv6 
  loopback addresses. #3725 #3726
- A `Publisher` or a `CompletableFuture` returned by a service now properly gets a cancellation signal when the
  request is cancelled. #3690 #3691
- Brave's `ScopeDecorator`s now correctly propagates a decorated `Scope` context when
  using [BraveService](type) and [BraveClient](type). #3408 #3430
- A [Server](type) in Spring integration is now automatically started by `SmartLifecycle`. #3759 #3762
- You no longer see [CancelledSubscriptionException](type) when a WebFlux's `WebClient` receives an unexpected
  `Content-Type` header. #3730 #3750
- [TomcatService](type) now properly handles an exception with Spring Web's `ExceptionHandler`s. #3447 #3732
- `ErrorWebExceptionHandler` in WebFlux converts an exception to the response correctly. #3779
- [JettyService](type) doesn't raise an `EofException` anymore. #3688
- Kotlin [`jsr305=strict`](https://kotlinlang.org/docs/java-interop.html#compiler-configuration) option no
  longer raises a false positive error when a null value is returned from where nulls are allowed
  to return. #3751
- You no longer see null values when a case class is decoded from a JSON when using annotated services with
  Scala. 400 Bad Request will be returned instead. #3728

## 🏚️ Deprecations

- `-Dcom.linecorp.armeria.annotatedServiceExceptionVerbosity` and [ExceptionVerbosity](type) has been
  deprecated. #3674
  - You can use [LoggingService](type) that automatically logs exceptions using for your convenience; or
    manually log exceptions using [ServerBuilder#exceptionHandler(ExceptionHandler)](type).
- [ObservableResponseConverterFunction#&lt;init&gt;(ResponseConverterFunction,ExceptionHandlerFunction)?full](type)
  has been deprecated in favor of
  [ObservableResponseConverterFunction#&lt;init&gt;(ResponseConverterFunction)?full](type). #3674
- [HttpResponseException#of(HttpStatus)?full](type) and [HttpResponseException#of(int)?full](type) has been
  deprecated in favor of [HttpStatusException](type). #3674

## ☢️ Breaking changes

- [ExceptionHandler](type) now returns [HttpResponse](type) instead of [AggregatedHttpResponse](type). #3674
  ```java
  // Before:
  ExceptionHandler handler = (ctx, cause) -> {
      if (cause instanceof IllegalArgumentException) {
          return AggregatedHttpResponse.of(HttpStatus.BAD_REQUEST);
      }
      return null;
  }
    
  // After:
  ExceptionHandler handler = (ctx, cause) -> {
      if (cause instanceof IllegalArgumentException) {
          return HttpResponse.of(HttpStatus.BAD_REQUEST); // 👈👈👈
      }
      return null;
  }
  ```
- [RequestContext](type) is added as the first parameter of [GrpcStatusFunction](type). #3692 #3693
  ```java
  // Before:
  GrpcService.builder()
             .exceptionMapping((throwable, metadata) -> {
                 if (throwable instanceof IllegalArgumentException) {
                     return Status.INVALID_ARGUMENT;
                 }
                 return null;
             });
  
  // After:
  GrpcService.builder()
             .exceptionMapping((ctx, throwable, metadata) -> { // 👈👈👈
                 if (throwable instanceof IllegalArgumentException) {
                     return Status.INVALID_ARGUMENT;
                 }
                 return null;
              });
  ```
- `JacksonModuleProvider` has been removed in favor of [JacksonObjectMapperProvider](type).
- `ResponseConverterFunctionProvider.createResponseConverterFunction(Type,ResponseConverterFunction,ExceptionHandlerFunction)`
  has been removed in favor of
  [ResponseConverterFunctionProvider#createResponseConverterFunction(Type,ResponseConverterFunction)?full](type)
  . #3674
- `OAuth2AuthorizationGrant.withAuthorization()` has been removed. #3618
