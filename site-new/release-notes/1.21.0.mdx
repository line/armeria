# v1.21.0

## 🌟 New features

- You can now use [@RequestTimeout](type) to set a request timeout to annotated services and
  gRPC services. #4463 #4499
  ```java
  import com.linecorp.armeria.server.annotation.decorator.RequestTimeout;
  
  @RequestTimeout(value = 1, unit = TimeUnit.SECONDS) // 👈👈👈
  class MyAnnotationService {
    // A request gets timed out after 1 second.
    @Get("/fast/api")
    public CompletableFuture<MyResponse> fastApi() {
      ...
    }

    // A request gets timed out after 5 seconds.
    @RequestTimeout(value = 5, unit = TimeUnit.SECONDS) // 👈👈👈
    public CompletableFuture<MyResponse> slowApi() {
      ...
    }
  }
  ```
- You can now set the default `MeterRegistry` by implementing a custom [FlagsProvider](type). #4541
  ```java
  public class CustomFlagsProvider implements FlagsProvider {
    @Override
    public MeterRegistry meterRegistry() {
      return PrometheusMeterRegistries.newRegistry(); // 👈👈👈
    }
  }
  ```
- You can now set the default [HttpHeaders](type) to [Server](type), [VirtualHost](type) and
  [Service](type). #4520
  ```java
  static AsciiString CUSTOM_HEADER =
    HttpHeaderNames.of("x-custom-header");

  Server
    .builder()
    .setHeader(CUSTOM_HEADER, "server-default-header")
    .virtualHost("foo.com")
    .setHeader(CUSTOM_HEADER, "vhost-default-header").and()
    .route()
    .path("/foo")
    .setHeader(CUSTOM_HEADER, "service-default-header")
  ```
- You can now set `:authority` or `Host` header whose value is different from [Endpoint#host()](type) using
  [ClientBuilder#setHeader(CharSequence,Object)](type). #4440 #4520
  - This will be useful when you want to send a request through a proxy server.
  ```java 
  WebClient
    // A request is sent to 'reverse-proxy.example.com'.
    .builder("http://reverse-proxy.example.com/")
    // but 'my-order.service.com' is used for the ':authority' header.
    .setHeader(HeaderNames.AUTHORITY, "my-order.service.com")
    ...
  ```
- [CircuitBreakerClient](type) can now be used with arbitrary [CircuitBreaker](type) implementations. #4426
- Added [Bytes](type) that represents binary data. #4505
- You can now see the description of services in the overview section of [DocService](type). #4480 #4537
- You can now override an authority of a gRPC client using `CallOptions`. #4445 #4520

## 📈 Improvements

- [DecodingClient](type) and [DecodingService](type) can now limit the maximum length of decompressed
  data. #4469 #4564
- `synchronized` blocks have been replaced with [ReentrantLock](type) for better virtual thread 
  compatibility. #4551 #4549
- [ServiceSpecification](type) for [DocService](type) is lazily loaded so as to not block a [Server](type)
  from starting. #4491

## 🛠️ Bug fixes

- [HttpHeaders](type) set by [ClientBuilder#setHeader(CharSequence,Object)](type) is now stored in
  [ClientRequestContext#defaultRequestHeaders()](type) which has the lowest priority and doesn't override
  values in [RequestHeaders](type).
  - Previously, the [HttpHeaders](type) set via [ClientBuilder](type) were stored in
    [ClientRequestContext#additionalRequestHeaders()](type) which overrode the values in
    [RequestHeaders](type) #4440 #4520
- Fixed a regression where `void` methods return `200 OK` instead of `204 NoContent` if a [@Produces](type)
  annotation is added. #4575
- Deleted uncompressed [DocService](type) assets that were added unintentionally. #4500 #4568
- An exception raised in [Authorizer](type) is now properly peeled using
  [Exceptions#peel(Throwable)](type). #4567
- Exceptions raised in decorators can now be used to determine whether to retry by [RetryingClient](type). #4558 #4559
- [EndpointGroup](type) is now correctly copied to a newly derived client. #4556
- [RequestContextExportingAppender](type) can now be used with logback `SocketAppender`. #4546 #4547
- Reversed HTTP/1.1 responses are now correctly ordered when HTTP/1.1 pipelining is enabled. #4536 #4470
- `414 Request-URI Too Long` is now correctly returned for a long URI in HTTP/1.1. #4478 #4532
- Fixed to propagate the cause of a decoder failure of an HTTP/1.1 request to
  [ServerErrorHandler#onProtocolViolation(ServiceConfig,RequestHeaders,HttpStatus,String,Throwable)](type) #4478 #4532
- You can use [DocService](type) even when [Server#reconfigure(ServerConfigurator)](type) is called. #4491 #4513
- Context leak stack trace is more readable now. #4517
